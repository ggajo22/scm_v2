# 인수 기준: SPEC-NAV-SIDEBAR-001

## 시나리오

---

### AC-001: "도서관리" 그룹 헤더 표시

**Given** 사용자가 인증된 상태로 애플리케이션에 접속해 있다  
**When** 사이드바가 렌더링된다  
**Then** "도서관리" 텍스트가 포함된 그룹 헤더가 화면에 표시된다  
**And** 해당 헤더는 `<a>` 태그나 클릭 가능한 링크가 아니어야 한다  
**And** `BookOpen` 아이콘이 헤더 옆에 표시된다  

**테스트 근거**: REQ-001

---

### AC-002: "도서관리" 하위 항목 링크 동작

**Given** 사용자가 인증된 상태로 있다  
**When** 사이드바가 렌더링된다  
**Then** "대시보드" 항목이 `/books` 경로로 연결되는 링크로 표시된다  
**And** "ISBN 추가" 항목이 `/books/add-isbn` 경로로 연결되는 링크로 표시된다  
**And** 두 항목 모두 "도서관리" 헤더 아래 들여쓰기된 위치에 표시된다  

**테스트 근거**: REQ-002, REQ-003, REQ-009

---

### AC-003: `/books` 경로에서 "대시보드" 활성 상태

**Given** 사용자가 인증된 상태다  
**When** 현재 경로가 정확히 `/books`다  
**Then** "대시보드" 항목에 활성 스타일(배경색 또는 강조 클래스)이 적용된다  
**And** "ISBN 추가" 항목에는 활성 스타일이 적용되지 않는다  

**테스트 근거**: REQ-004

---

### AC-004: `/books/add-isbn` 경로에서 "ISBN 추가" 활성 상태

**Given** 사용자가 인증된 상태다  
**When** 현재 경로가 `/books/add-isbn`다  
**Then** "ISBN 추가" 항목에 활성 스타일이 적용된다  
**And** "대시보드" 항목에는 활성 스타일이 적용되지 않는다  

**테스트 근거**: REQ-005

---

### AC-005: `/books/:id` 경로에서 어떤 하위 항목도 활성 상태 아님

**Given** 사용자가 인증된 상태다  
**When** 현재 경로가 `/books/123` (도서 상세 페이지)이다  
**Then** "대시보드" 항목에 활성 스타일이 적용되지 않는다  
**And** "ISBN 추가" 항목에도 활성 스타일이 적용되지 않는다  

**테스트 근거**: REQ-006

---

### AC-006: `super_admin`에게 "관리자 계정 관리" 항목 표시

**Given** 인증된 사용자의 역할이 `super_admin`이다  
**When** 사이드바가 렌더링된다  
**Then** "관리자 계정 관리" 항목이 최상위 플랫 항목으로 표시된다  
**And** 해당 항목은 `/admin-users`로 연결된다  
**And** 해당 항목은 "도서관리" 그룹의 하위 항목이 아닌 독립 항목으로 표시된다  

**테스트 근거**: REQ-007

---

### AC-007: `admin`에게 "관리자 계정 관리" 항목 미표시

**Given** 인증된 사용자의 역할이 `admin`이다 (super_admin 아님)  
**When** 사이드바가 렌더링된다  
**Then** "관리자 계정 관리" 항목이 화면에 표시되지 않는다  

**테스트 근거**: REQ-008

---

### AC-008: 그룹 헤더 접근성 속성

**Given** 사이드바가 렌더링된다  
**When** 보조 기술(스크린 리더)이 "도서관리" 그룹 영역을 탐색한다  
**Then** 그룹 헤더 요소에 `aria-label="도서관리"` 또는 동등한 접근성 속성이 존재한다  
**And** 그룹 헤더 자체는 포커스 가능한 대화형 요소가 아니다  

**테스트 근거**: REQ-010
